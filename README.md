# ひらがな よもう

自分で書いたひらがなを、AI（端末の音声）が声に出して読んでくれる練習アプリ。
**「合ってる／間違ってる」に、子ども本人が音で気づける**ことを目的にしています。
ダウン症のお子さん（ひらがなを少し読める・書ける、正確性はこれから）に向けて設計しました。

---

## このアプリの芯（壁打ちで決めた方針）

- **主役は「自己フィードバック型の学習」**。大人が「違うよ」と直すのではなく、音と色で本人が気づく。
- **間違いは正直に読む**。「ねこ」のつもりで「ねご」と打ったら、まず正直に「ねご」と読む → 本人が耳で気づく。
- **責めない添削**。間違った文字だけ *そっと黄色で囲む*（合ってる文字には何もしない）。
- **最後に選ぶのは本人**。「ここ『こ』にする？ それとも このまま『ご』？」と問いかけ、本人が選ぶ。
- **できたら思いっきり祝う**。正解で大きな花丸＋「ピンポン♪」。書けた字は隠さず見せる。

---

## 2つのモード

| モード | ねらい | ふるまい |
|---|---|---|
| **おだい** | しっかり添削してもらう | お題（例「ねこ」）と照合。違う字を黄色で囲み、二択で問いかけ→選ぶ→正解で花丸 |
| **じゆう** | のびのび書いて音で確かめる | 打った通りに正直に読み上げるだけ。正誤の断定はしない |

## 入力まわりの仕様

### キーボード（自作の五十音グリッド）
- iPad のかなキーボードに近い配列。**右端の列が縦に あ・い・う・え・お**、列は右→左へ あ行→か行→…→わ行。
- **左端の1列は機能キー**：上から「**゛（てんてん）→ ゜（まる）→ ー（のばし棒）**」。よく使う順。
- 機能キーの下に「**きごう ▼**」トグル。ふだんは隠れていて、押すと「、。？！」が出る（子どもが迷わないように）。
- ローマ字・フリックは使わない（覚え始めの子に負荷が高いため）。

### 濁点・半濁点
- 文字を打ってから「゛」「゜」を押すと、**直前の1文字**に付く（き→ぎ、は→ぱ）。
- もう一度押すと戻せる（ぎ→き）。付けられない文字（あ・ん など）は何も起きない。
- 変換表は `lib/kana.js` の `DAKUTEN` / `HANDAKUTEN`。

### 単語から文へ
- 「ねこ」と打って「よむ」→ 続けて「がすき」と打って「よむ」、と区切って積み上げられる。
- 画面は**ひらがなのまま**、音だけ自然に読む（漢字変換はしない＝学習中の混乱を避ける）。

## おうちの人へ（設定）

画面下の「⚙ おうちの人へ」を開くと：
- **お題の追加**：お子さんの名前・家族・好きなキャラ・好きな食べものなどを入れられる（覚え始めの定番は最初から入っている）。
- **お題の一覧／削除**：タップでそのお題に切替、×で削除。
- **声の速さ**：スライダーで調整。
- 設定（お題・速さ）は**この端末に保存**され、次回も残る（`localStorage`）。

---

## 技術構成

- **Next.js 14（App Router）+ React 18**。
- 読み上げ：ブラウザの **Web Speech API**（`SpeechSynthesis`）。追加インストール不要・オフライン可。iPad は Safari 推奨。
- 「ピンポン♪」：**Web Audio API** で生成（音声ファイル不要）。
- データ保存：**localStorage**（サーバー不要）。
- PWA 対応の下地（`manifest.json`）あり。ホーム画面に追加するとフルスクリーンで使える。

### ファイル構成
```
hiragana-yomou/
├─ app/
│  ├─ layout.jsx        # フォント読込・viewport（ピンチズーム無効）
│  ├─ page.jsx          # トップページ
│  └─ globals.css       # リセット・タッチ最適化
├─ components/
│  ├─ HiraganaApp.jsx   # 画面本体（状態・UI・スタイル）
│  └─ Graphics.jsx      # マスコット・花丸の SVG
├─ lib/
│  ├─ kana.js           # 配列・濁点変換・お題・判定ロジック（★中身はここを編集）
│  └─ useSpeech.js      # 読み上げ＆チャイムのフック
├─ public/
│  └─ manifest.json     # PWA 設定（アイコン icon-192.png / icon-512.png は要追加）
├─ package.json
├─ jsconfig.json        # @/ パスエイリアス
└─ next.config.mjs
```

「**中身（配列・お題・濁点・色など）を変えたいときは主に `lib/kana.js`**、見た目を変えたいときは `components/HiraganaApp.jsx` の `S`（スタイル）と `C`（色）」という分け方にしています。

## 起動方法

```bash
cd hiragana-yomou
npm install
npm run dev
# → http://localhost:3000 をブラウザで開く
```

iPad の実機で試すには、PC と iPad を同じ Wi-Fi につなぎ、`npm run dev` 起動後に表示される
ネットワークURL（例 `http://192.168.x.x:3000`）を iPad の Safari で開きます。

## 今後の宿題（壁打ちで「あとで」にしたもの）

1. **字数がずれる間違いへの対応**：いまは「ねっこ」のように字数が違うと断定せず「おしい！」とだけ伝える。
   将来、辞書照合や近い言葉の推測で、より踏み込んだ添削を検討。
2. **文の正誤チェック**：いまは単語単位の添削。「がっこうへいく」が文として正しいか、は将来のステップ。
3. **1文字ずつの音**：「あ」を押した瞬間に「あ」と鳴らす（音と文字の結びつき強化）モードの追加。
4. **なぞり書き機能（第2弾）**：お手本をなぞって「書く」運動の練習。タイピングとは別画面の作りになる想定。
5. **設定の保護**：「おうちの人へ」を長押しや隠しジェスチャーの奥に置き、子どもが誤って触らないようにする。
6. **自由モードの問いかけ**：自由入力で「これであってるかな？」とそっと促す挙動（断定はしない）。
7. **PWA アイコン**：`public/icon-192.png` / `icon-512.png` を用意（ホーム画面追加用）。
